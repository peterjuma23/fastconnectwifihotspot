# FastConnect Internet — WiFi Billing System

Production-ready WiFi hotspot billing system for Kenya. Built for ISPs and cafes serving hundreds of concurrent users.

**Stack:** Node.js · React · MySQL 8 · Redis · MikroTik RouterOS · M-Pesa Daraja API · Docker · Nginx

---

## Features

- **M-Pesa STK Push** — customers pay directly from their phone, no app needed
- **Real-time plan updates** — admin edits reflect on portal instantly via WebSocket
- **Per-device credentials** — each phone+browser gets unique hotspot username/password
- **Auto-reconnect** — returning users reconnect automatically, no re-purchase needed
- **Voucher system** — generate bulk codes (FC-XXXX-XXXX) for resellers or promos
- **Multi-router** — manage multiple MikroTik hotspots from one dashboard
- **2FA admin login** — TOTP via Google Authenticator
- **Audit logging** — every admin action, payment, and plan change logged
- **Automated backups** — encrypted daily MySQL dumps

## Quick Start (Production)

### 1. Server Requirements

- Ubuntu 22.04 LTS (or Debian 12)
- 2 vCPU, 4 GB RAM minimum
- Ports 80 and 443 open
- Domain DNS pointing to server: `wifi.fastconnect.co.ke`, `admin.fastconnect.co.ke`

### 2. Install Dependencies

```bash
# Docker + Docker Compose
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER && newgrp docker

# Make, Node (for key generation)
sudo apt install -y make nodejs
```

### 3. Clone & Configure

```bash
git clone https://github.com/your-org/fastconnect.git
cd fastconnect
make setup
```

Edit `.env` with your values:

```bash
nano .env
```

**Required values to fill in:**
- `DB_ROOT_PASSWORD`, `DB_PASSWORD` — choose strong passwords
- `MPESA_CONSUMER_KEY`, `MPESA_CONSUMER_SECRET` — from Safaricom Developer Portal
- `MPESA_PASSKEY` — from Safaricom (Lipa Na M-Pesa passkey)
- `MPESA_SHORTCODE` — your paybill or till number
- `JWT_PRIVATE_KEY`, `JWT_PUBLIC_KEY` — generated by `make generate-keys`
- `ENCRYPTION_KEY` — 64-char hex, generated by `make generate-keys`

### 4. Obtain SSL Certificates

```bash
# First, start Nginx on HTTP only for ACME challenge
make start

# Then get certs
make ssl
```

### 5. Build & Launch

```bash
make build
make start
make migrate
make seed
make health-check
```

### 6. Verify

- Portal: `https://wifi.fastconnect.co.ke`
- Admin: `https://admin.fastconnect.co.ke`
- Default admin: `admin` / `Admin@FastConnect2024!`
- **Change the admin password immediately after first login**

## M-Pesa Setup

1. Register at [Safaricom Developer Portal](https://developer.safaricom.co.ke)
2. Create an app and get Consumer Key + Consumer Secret
3. Go to **Lipa Na M-Pesa Online** to get your Passkey
4. Set callback URL: `https://wifi.fastconnect.co.ke/api/payments/mpesa-callback`
5. Start with `MPESA_ENVIRONMENT=sandbox` for testing
6. Switch to `production` when ready to go live

## MikroTik Router Setup

### On the Router (via Winbox or SSH):

```
# Enable SSH API
/ip service enable ssh

# Create hotspot on your LAN interface
/ip hotspot setup

# Create a hotspot user profile (must match HOTSPOT_PROFILE in .env)
/ip hotspot user profile add name=fastconnect rate-limit=2M/2M

# Create API user for FastConnect
/user add name=fastconnect-api password=<strong-password> group=full
```

### Add Router in Admin Dashboard:

1. Go to Admin → Routers → Add Router
2. Fill in: IP address, SSH port (22), username, password
3. Click "Test Connection" to verify
4. Set hotspot profile name (must match what you created above)

## Admin Guide

### Managing Plans

1. **Edit a plan:** Admin → Pricing Plans → Edit
   - Changes apply instantly to the portal (WebSocket broadcast)
   - Active sessions keep their original price/duration (stored in `plan_snapshot_json`)
   - A history entry is saved in `plan_history` table

2. **Mark as Popular:** Click the star icon — shows "⭐ Popular" badge on portal
   - Only one plan can be popular at a time

3. **Delete a plan:** Only allowed if no active sessions use it
   - Soft delete: plan remains in DB for audit purposes

4. **Reorder plans:** Drag the ⠿ handle to reorder — drag-and-drop saves automatically

### Generating Vouchers

1. Admin → Vouchers → Generate Vouchers
2. Choose plan, quantity (up to 1000), validity (days)
3. Export CSV to print or distribute to resellers
4. Codes are in `FC-XXXX-XXXX` format

### Force Disconnecting Users

Admin → Active Users → Disconnect button

This:
- Removes user from MikroTik hotspot immediately
- Marks session as `disconnected` in database
- Logs the action in audit trail
- Does **not** refund payment

### Viewing Audit Logs

Admin → Audit Logs

All logged events:
- Admin logins (with IP)
- Plan creates / edits / deletes (with before/after values)
- Payment completions
- Session disconnects
- Voucher generation and redemption
- Router health check failures

## Security

- **HTTPS everywhere** — TLS 1.2+ with HSTS preload
- **RS256 JWT** — asymmetric signing, 15-min expiry
- **Session binding** — tokens bound to IP + User-Agent
- **2FA** — TOTP for all admin accounts
- **AES-256 encryption** — phone numbers, device IDs, router passwords at rest
- **Parameterized queries** — all SQL uses prepared statements
- **Rate limiting** — Nginx + Express (global, login, payment, admin)
- **Input validation** — express-validator on all endpoints
- **CSP headers** — strict Content-Security-Policy
- **No PII leakage** — error messages never contain stack traces in production

## API Reference

Full OpenAPI 3.0 spec: `docs/openapi.yaml`

Import into [Postman](https://postman.com) or view with [Swagger UI](https://editor.swagger.io).

### Key Endpoints

```
GET  /api/plans                           List active plans (cached)
GET  /api/sessions/status/:phone          Check session for device
POST /api/payments/initiate               Start M-Pesa STK Push
GET  /api/payments/status/:id             Poll payment status
POST /api/payments/mpesa-callback         M-Pesa webhook
POST /api/vouchers/redeem                 Redeem voucher code

POST /api/admin/auth/login                Admin login (rate limited)
GET  /api/admin/dashboard/stats           Live dashboard stats
GET  /api/admin/sessions/active           Active user list
POST /api/admin/sessions/:id/disconnect   Force disconnect user
GET  /api/admin/plans                     All plans (including inactive)
POST /api/admin/plans                     Create plan
PUT  /api/admin/plans/:id                 Update plan (preserves active sessions)
DELETE /api/admin/plans/:id              Soft delete plan
POST /api/admin/vouchers/generate         Generate bulk vouchers
GET  /api/admin/reports/sales             Sales reports
```

### WebSocket

Connect to `wss://wifi.fastconnect.co.ke/ws/plans` for real-time plan updates.

Messages received:
```json
{ "action": "PLAN_UPDATED", "plan": {...}, "timestamp": 1700000000000 }
{ "action": "PLAN_CREATED", "plan": {...}, "timestamp": 1700000000000 }
{ "action": "PLAN_DELETED", "plan": { "id": 4 }, "timestamp": 1700000000000 }
```
